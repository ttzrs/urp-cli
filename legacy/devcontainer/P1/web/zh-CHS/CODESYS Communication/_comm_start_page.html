CODESYS Communication

CODESYS Communication 允许在应用程序中简单地发布和链接数据。

数据通过 OPC UA 发布。 Communication Manager 功能为此提供了两个选项：

  • 使用 OPC UA 信息模型

    这些都是普遍支持的。信息模型的内容被翻译成相应的 IEC 数据类型和声明。

  • 使用 IEC 符号组配置 通讯经理的

    IEC 符号可以快速、轻松地发布，无需事先明确创建信息模型。

新的 Communication Manager 功能为数据发布提供了以下优势：

  • 您可以将自己的信息模型用于 OPC UA。

  • 可以使用信息模型调用方法。

  • 您可以定义完全独立的符号集。

    在前面的符号配置中，可以根据符号集隐藏全局变量，但类型配置始终必须相同。

  • 应用程序和信息模型之间的附加抽象

    在里面 通讯经理，客户端的名称可以单独配置，从而实现完全分离。

    相反，在之前的符号配置中，符号的 IEC 名称也向外传输，这意味着应用程序中的重构可能会导致客户端必须进行调整。

  • 简化符号集的权限管理

    有关更多信息，请参阅： IEC符号出版

  • 更好的工作流程：IEC符号发布编辑器提供应用程序的预编译信息以进行配置。这意味着项目中可用的变量和类型是为了在编辑器中配置符号而提供的，而无需无错误地编译应用程序。由于项目变更可能导致编辑与项目之间的差异，我们将提请您注意，并协助您进行更正。

  • 符号集标签计数的详细输出

    用户可以更好地了解哪些符号被有效释放。

  • 用户友好的编辑器，例如用于提高清晰度的过滤选项

数据是通过数据源接收的。源可以是另一个 CODESYS 应用程序或任何 OPC UA服务器。

你可以 CODESYS Communication 在里面 CODESYS Store International 或在 CODESYS Store North America 下载。在那里您还可以找到该产品的数据表。

CODESYS Scripting

CODESYS Scripting 允许您自动执行命令或复杂的程序操作 CODESYS，否则需要手动单击鼠标并在 CODESYS 用户界面。为此，基于 (Iron)Python 的脚本可以从 CODESYS 用户界面（命令或配置的工具栏）或从 Windows 命令行。
