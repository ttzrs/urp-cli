CODESYS Communication

CODESYS Communication アプリケーションでのデータの簡単な公開とリンクを可能にします。

データは OPC UA 経由で公開されます。 Communication Manager 機能には、これに対して 2 つのオプションが用意されています。

  • 使用する OPC UA 情報モデルレン

    これらは一般的にサポートされています。情報モデルの内容は、対応する IEC データ型と宣言に変換されます。

  • の使用 IECシンボルセット構成 コミュニケーションマネージャーの

    IEC シンボルは、事前に情報モデルを明示的に作成しなくても、迅速かつ簡単にリリースできます。

新しい Communication Manager 機能には、データの公開に関して次の利点があります。

  • OPC UA には独自の情報モデルを使用できます。

  • 情報モデルを使用したメソッド呼び出しが可能です。

  • 完全に独立したシンボル セットを定義できます。

    前のシンボル設定では、シンボル セットに応じてグローバル変数を非表示にすることができますが、型設定は常に同一である必要があります。

  • アプリケーションと情報モデルの間の追加の抽象化

    の中に コミュニケーションマネージャー、クライアントの名前を個別に構成できるため、明確に分離できます。

    対照的に、前のシンボル構成では、シンボルの IEC 名も外部に転送されます。これは、アプリケーションでのリファクタリングにより、クライアントを適応させる必要がある可能性があることを意味します。

  • シンボルセットの権利管理の簡素化

    詳細については、以下を参照してください。 IEC シンボルの発行

  • ワークフローの改善: IEC シンボル パブリケーション エディターは、設定用のアプリケーションのプリコンパイル情報を提供します。これは、アプリケーションをエラーなしでコンパイルする必要がなく、プロジェクトで使用可能な変数と型がエディターでのシンボルの構成に提供されることを意味します。プロジェクトの変更によって生じる可能性のある編集者とプロジェクトとの間の矛盾には注意が払われ、修正を支援します。

  • シンボルセットのタグ数の詳細な出力

    ユーザーは、どのシンボルが効果的に解放されるかをよりよく理解できます。

  • ユーザーフレンドリーなエディター (例: 明瞭さを向上させるフィルター オプション)

データはデータ ソース経由で受信されます。ソースは別のものでもかまいません CODESYS アプリケーションまたは任意の OPC UAサーバー。
