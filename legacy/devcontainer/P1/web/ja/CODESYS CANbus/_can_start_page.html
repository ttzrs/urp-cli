CODESYS CANbus

CODESYS 次のCANベースのフィールドバスをサポートします。

  • 開けられる

  • J1939

フィールドバスのサポートに関する一般的な情報については、CODESYS、 見る： フィールドバスのサポート

開けられる

CANopenは、フローの自動化に使用されるプロトコルです。 CANopenネットワークは、マスターとスレーブとして機能するマネージャーとデバイスで構成されます。この場合、デバイスはモジュール式または非モジュール式に構造化できます。

さらに、 CODESYS リモートデバイスとローカルデバイスを区別します。

  • CANopenリモートデバイス： の CODESYS、CANopenリモートデバイスは、プロジェクトのデバイスツリーのCANopenマネージャー（マスター）の下に挿入されるスレーブデバイスです。

  • CANopenローカルデバイス： の CODESYS CANopenローカルデバイスは、プロジェクトのデバイスツリーのCANバスノードのすぐ下に挿入されるスレーブデバイスです。その結果、 CODESYS ランタイムはCANopenスレーブとして使用できます。

ダウンロードできます CODESYS CANopen Manager SL の中に CODESYSストア.そこには、製品のデータシートもあります。

注記

CANopenサンプルプロジェクト

J1939

ネットワークプロトコルSAE1939は、CANを物理的なトランスポート層としても使用します。このプロトコルは、商用車のコントローラー情報と診断データを転送するために使用されます。

ダウンロードできます CODESYS J1939 Manager SL の中に CODESYSストア.そこには、製品のデータシートもあります。
