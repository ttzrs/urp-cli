Static AnalysisÂ¶

Note

The CODESYS Professional Developer Edition provides a tool for static code analysis. A rule set is included for automatically checking the naming convention described above.

Under some circumstances it would be necessary to deactivate a specific rule. CODESYS Static Analysis provides pragmas and attributes for activating or deactivating individual rules or naming conventions for static code analysis.

Attributes are inserted in the declaration part of a POU.

{attribute 'analysis' := '-33, -31'}
/// ...
/// some code which does not conforme to rule SA0031 and SA0033
/// ...
{attribute 'analysis' := '+33, +31'}

Pragmas are used in the implementation part of a POU.

{analysis -46}
/// ...
/// some code which does not conforme to rule SA0046
/// ...
{analysis +46}
